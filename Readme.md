# ⚡ AI-Powered Code Reviewer

A full-stack web application built using Next.js, Express, Node.js, and MongoDB that acts as an automated Senior Developer. Users can write or paste source code into a custom web sandbox, run a live review pipeline powered by Google Gemini AI, and safely track their optimization audit logs inside a dedicated history dashboard.

---

## 🏗️ System Architecture & Data Flow

The layout below illustrates how data seamlessly traverses the stack, safely processing the code through external AI pipelines before transforming and storing the results into the cloud database layer.

1. **Client Workspace (Frontend):** The user writes code inside a custom, Monaco-free text interface. When they execute a review, an Axios request sends the source code string to the server.
2. **Review Gateway (Backend):** The Express engine receives the payload, verifies authentication, and pipes the code context directly to the Google Gemini API using strict JSON enforcement parameters.
3. **AI Telemetry & Transformation:** The Gemini model parses the code framework, compiles a structured JSON array containing isolated issue tokens and a clean refactored code block patch, and responds back to the backend server.
4. **Cloud Cluster Pipeline (Database):** The backend saves the immutable code snapshot alongside the full, flexible AI array into a MongoDB collection using an optimized update query, then hydrates the dual-split UI layout with the complete review data.

## 🛠️ Tech Stack & Core Dependencies

### Frontend (User Interface)
- **Framework:** Next.js (App Router architecture with reactive Client Components)
- **Styling UI:** Tailwind CSS (Engineered with deep dark-mode themes for reduced eye strain)
- **Network Client:** Axios (Handles asynchronous API handshakes with interceptors)

### Backend (Server Layer)
- **Runtime Environment:** Node.js with Express web framework
- **Database Engine:** MongoDB & Mongoose ODM (Object Data Modeling)
- **AI Core Client:** Google Gemini API (`gemini-2.5-flash` intelligence model)

## ✨ Core Features

### 🧠 Structured AI Response Engine
- Forces the Gemini API to respond **only** in a strict, organized JSON array by using the backend parameter `responseMimeType: "application/json"`.
- Prevents the AI from outputting messy, markdown text blocks that are difficult for websites to parse and display.
- Every detected bug, performance block, or security threat is safely delivered with an explicit category, title, deep explanation text, and severity rating.

### 🛠️ Dynamic Data Hydration (`Mixed` Schema Type)
- Overcomes the limitation where strict database schemas strip out dynamic fields generated by AI systems, ensuring no data loss during cloud synchronization.
- Utilizes Mongoose's `Schema.Types.Mixed` data configuration within the MongoDB history schema layer to store unpredictable JSON structures safely.
- Allows the database to save and reconstruct everything Gemini outputs, matching the structure of the AI array exactly.

### 🛡️ Graceful Degradation & High Resilience
- The backend separates live AI operations from database persistence tasks.
- If your MongoDB database encounters an unexpected cloud cluster disconnect or network error, the server catches the exception gracefully.
- The pipeline never crashes; it still displays the live AI review text right on the user's screen while informing them that the session couldn't be logged to history.

### 🗄️ Synchronized Dual-View Archive
- Features a dedicated review history dashboard page.
- Clicking on an archived session card instantly hydrates the view state, allowing developers to inspect the original code snippet and its corresponding AI findings side-by-side.
- Reloads historical results instantly from the database without invoking secondary, expensive API endpoint requests to Gemini.
